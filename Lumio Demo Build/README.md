# Lumio — Client Demonstration Build

This folder is a **standalone, static demonstration build** of the Lumio prototype.
It is built with plain HTML, CSS, and JavaScript — there is no build step, bundler,
or server-side component required.

> **This is not a production release.** It is a demonstration build for client
> review. See "Current Development Roadmap" below for features that are still
> in progress.

---

## 1. What's included

- **Projects, Recent, Trash** — the workspace dashboard with 7 sample projects
- **ID Academy** — 9 instructional-design learning paths
- **Course Wizard** — guided course creation flow (Help Me Decide, Theme Designer, AI suggestions)
- **Course Landing Page** — editable course landing pages with 5 layout options
- **Theme Engine** — colour palettes, fonts, button styles, corner radii, and page backgrounds
- **Hero Image System** — configurable hero banners with overlays, positioning, and text styling
- **Landing Page Sections** — additional configurable landing page content blocks
- **Lesson Builder** — drag-and-drop block editor with 12 block categories
- **Text & Statement Blocks** — paragraphs, headings, columns, tables, and 6 statement types (info, tip, success, warning, error, note)
- **Drag-and-Drop & Block Reordering** — fully interactive in the Lesson Builder
- **Learner Preview** — Desktop / Tablet / Mobile responsive preview modes
- **Course Settings** — theme, hero, and landing configuration per course

### Demonstration dataset

Three fully-populated demo courses, each with a distinct theme, hero layout,
landing page configuration, and lesson structure:

| Course | Theme | Landing Layout | Page Background |
|---|---|---|---|
| **New Hire Onboarding** | Indigo Focus (Poppins + Inter, Pill buttons) | A — Centered | Soft Gradient |
| **Workplace Safety Basics** | Teal Calm (Montserrat + Nunito, Square buttons) | B — Text Left / Image Right | Light Grey |
| **Formula 1 Fundamentals** | Sunset (Space Grotesk, Rounded buttons) | C — Full Banner | White |

Each course has 3 lessons built from a mix of Text, Statement, List, Table,
Accordion, and Knowledge Check blocks.

## 2. Hosting locally

Because the app uses plain `<script src="...">` tags (no `fetch`/CORS-sensitive
calls), you can open `index.html` directly in a browser by double-clicking it.

For the most reliable experience (and to match how it will behave on a real
host), serve it from a simple static file server:

**Using Python (built-in on most systems):**
```bash
cd "Lumio Demo Build"
python -m http.server 8080
```
Then open `http://localhost:8080` in your browser.

**Using Node.js:**
```bash
cd "Lumio Demo Build"
npx serve .
```

**Using VS Code:**
Install the "Live Server" extension, right-click `index.html`, and choose
"Open with Live Server".

## 3. Hosting on Cloudflare Pages

This is a static site with no build step.

1. Push this folder (or the whole repo) to a GitHub/GitLab repository.
2. In the Cloudflare dashboard, go to **Workers & Pages → Create → Pages → Connect to Git**.
3. Select the repository.
4. Configure the build settings:
   - **Framework preset:** None
   - **Build command:** *(leave empty)*
   - **Build output directory:** `Lumio Demo Build` (or `/` if this folder is the repo root)
5. Click **Save and Deploy**.

No environment variables, databases, or backend services are required.

## 4. Known limitations

- **No persistence between browsers/devices** — all data is stored in the
  browser's `localStorage`. Clearing browser storage resets the demo to the
  seeded dataset above.
- **No real backend** — there is no authentication, database, or file storage.
  Login is a visual flow only; any credentials are accepted.
- **Hero images use placeholder gradients** — the Hero Image system supports
  uploading real images, but the demo courses use the theme-colour gradient
  fallback rather than uploaded photography.
- **AI features are simulated** — "AI Assistant" responses in the Lesson
  Builder and Wizard are pre-written, not generated by a live model.
- **Single learner profile** — Learner Preview tracks progress for one
  simulated learner per course; there is no multi-learner reporting.

## 5. Current Development Roadmap

The following areas are visible in the build but are still under active
development. They have been left in place (not removed) so the client can see
the overall direction and current state of progress:

- **Assessments module** — assessment items appear in course structure and
  navigation, but the full assessment-authoring experience is still being
  built out beyond the Lesson Builder's knowledge-check blocks.
- **Additional block types** — categories such as Charts, Multimedia, and
  Gallery are listed in the block library; some are partially implemented.
- **Course publishing / sharing** — "Publish" and sharing/export flows are
  represented in the UI but are not yet functional end-to-end.
- **Multi-user / collaboration** — the workspace currently reflects a single
  user (Workspace Admin); team roles and permissions are planned.
- **Reporting & analytics** — learner completion data is tracked locally for
  the preview but not surfaced in any reporting dashboard yet.
